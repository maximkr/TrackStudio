# Расширение и переопределение ролей

Иногда требуется, чтобы в одном проекте пользователь был и разработчиком, и тестировщиком - требуется расширить права пользователя. В таком случае его нужно еще раз добавить в правила доступа, а затем в одной из записей изменить группу:

![](../../../images/screenshot9.png)

Если просто добавить пользователя в права доступа и назначить ему другую роль, у него будет две роли: его собственная (та, что соответствует его учетной записи) и переопределенная:

![](../../../images/screenshot11.png)

![](../../../images/screenshot10.png)

Для того, чтобы оставить только одну роль тестировщика, нужно выбрать опцию **Переопределен**

![](../../../images/screenshot13.png)

Тот же принцип работает для правил доступа, определенных на разных уровнях: на нижестоящем уровне можно переопределить роль, заданную на вышестоящем.
