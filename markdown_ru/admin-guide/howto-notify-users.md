# Как оповещать пользователей об изменениях по e-mail

Пользователи в TrackStudio могут получать оповещения по почте о новых задачах или об изменении существующих, а также получать периодические уведомления по подписке.

Для этого нужно включить рассылку оповещений и уведомлений в Server Manager. Укажите адрес SMTP-сервера и данные почтовой учетной записи, если для отправки требуется авторизация. Также укажите обратный адрес.

![](../images/enable-email-notification-ru.png)

![](../images/gmail-smtp.png)

Каждое оповещение имеет точку приложения, задачу, с которой оно связано. Правила оповещения распространяются на саму задачу и ее подзадачи и определяются выбранным фильтром. Если задача удовлетворяет условию фильтрации и в оповещении выбрано соответствующее событие - оповещение будет отправлено. Вы можете получать оповещения по следующим событиям: кто-то создал новую задачу, кто-то отредактировал существующую задачу, кто-то приложил к задаче файл, кто-то выполнил в задаче операцию (например, закрыл задачу или переоткрыл).

![](../images/event-notification-ru.png)

Для того, чтобы подписаться на уведомления, перейдите в ваш проект или на нужную вам задачу и выберите пункт меню "Оповещения о событиях". Затем нажмите кнопку "Создать оповещение о событиях для себя". Выберите нужный фильтр и один из почтовых шаблонов. Отметьте события, по которым вы хотите получать оповещения и нажмите кнопку "Сохранить".

Так же вы можете создавать оповещения для своих подчиненных. В этом случае вам нужно будет выбрать пользователя, либо роль - тогда все ваши подчиненные с этой ролью, у кого есть доступ к задаче, будут получать оповещения.

Для проверки оповещений используется кнопка "Проверить", которая находится на странице просмотра оповещения. Здесь нужно учитывать, что тестовое оповещение будет отправлено, если задача, в которой создано оповещение, попадает в фильтр, выбранный в оповещении.
