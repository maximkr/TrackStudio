# Как настроить права для пользователей на операции и дополнительные поля в процессах

Вы можете настроить права пользователей с определенной ролью [на процессы](../workflow/workflow-operations/workflow-operation-permissions.md) (выполнение операций для этих процессов) и [для дополнительных полей](../workflow/workflow-udf.md).

![](../../images/roles_process_permissions.png)

## Права на операции

Вы можете настроить правила, по которым пользователи с данной ролью смогут или не смогут видеть операции, выполнять их и становиться ответственными за задачу на определенных этапах ее решения.

- Перейдите через меню к управлению пользователями
- Перейдите через меню к странице "**Роли**"
- Перейдите к нужной роли через список
- Выберите вкладку "**Разрешения по процессам**"
- Выберите нужный процесс из списка
- Выберите вкладку "**Права на операции в зависимости от роли**"

Настройки каждого из правил представлены в виде пар списков операций. Пользователи с данной ролью не могут выполнять действия из левого списка и могут — из правого.

![](../../images/roles_operations_permissions.png)

При создании операции автоматически выставляются разрешительные права для всех ролей, к которым есть доступ у пользователя, создавшего операцию. При создании новой роли такие разрешения **не выставляются**.

Настройки правил не должны противоречить друг другу. Например, нельзя разрешить выполнение операции, но не разрешить ее просмотр.

Для каждой операции из списка разрешенных вы также можете указать определенные модификаторы. Так, например, вы можете разрешить выполнение операции не всем менеджерам, а только **Менеджеру**, являющемуся автором или ответственным задачи. Для того, чтобы использовать модификатор, выберите нужную операцию (или операции) из списка и нажмите кнопку модификатора. Вы можете использовать два модификатора (**Автор** и **Ответственный**), если они доступны. Пары модификаторов работают по **ИЛИ**.

Модификаторы действуют для всех правил одинаково, то есть **Автор** — это автор задачи, а **Ответственный** — ответственный за нее на момент проверки прав.

## Права на дополнительные поля

- Перейдите через меню к управлению пользователями
- Перейдите через меню к странице "**Роли**"
- Перейдите к нужной роли через список
- Выберите вкладку "**Разрешения по процессам**"
- Выберите нужный процесс из списка
- Выберите вкладку "**Права на дополнительные поля в зависимости от роли**"

Настройки каждого из правил представлены в виде пар списков полей. Пользователи с данной ролью не могут редактировать или смотреть поля из левого списка и могут — из правого.

![](../../images/roles_workflow_udf_permission.png)

Настройки правил не должны противоречить друг другу. Например, нельзя разрешить выполнение операции, но не разрешить ее просмотр.

Для каждого поля из списка разрешенных вы также можете указать определенные модификаторы. Так, например, вы можете разрешить просмотр поля не всем менеджерам, а только **Менеджеру**, являющемуся автором или ответственным задачи. Для того, чтобы использовать модификатор, выберите нужное поле (или поля) из списка и нажмите кнопку модификатора. Вы можете использовать два модификатора (**Автор** и **Ответственный**), если они доступны. Пары модификаторов работают по **ИЛИ**.

Модификаторы действуют для всех правил одинаково, то есть **Автор** — это автор задачи, а **Ответственный** — ответственный за нее на момент проверки прав.
