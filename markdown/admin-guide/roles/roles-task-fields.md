# Как запретить просмотр или редактирование полей в задачах

В TrackStudio вы можете гибко управлять видимостью и доступностью для редактирования полей в задачах для каждой роли отдельно. Для того, чтобы настроить права на поля в задачах:

- Перейдите через меню к управлению пользователями
- Перейдите через меню к странице "**Роли**"
- Перейдите к нужной роли через список
- Выберите вкладку "**Доступ к полям задач**"

![](../../images/roles_task_fields.png)

На этой вкладке представлена таблица с полями задач, а также списки дополнительных полей, привязанных к задачам (но не к процессам).

Для некоторых стандартных полей в задачах вы можете управлять и видимостью и редактированием (*Приоритет, Сделать до, Бюджет, Потрачено времени и Описание*). Для некоторых только редактированием (*Код, Ответственный*), а для некоторых — только видимостью (*Резолюция, Дата создания, Дата обновления, Дата закрытия*)

Вы можете управлять видимостью и редактированием дополнительных полей, привязанных к задачам. Настройки каждого из правил представлены в виде пар списков полей. Пользователи с заданной ролью могут видеть и редактировать поля из правых столбцов и не могут — из левых.

Настройки правил не должны противоречить друг другу. Например, нельзя разрешить редактирование поля, но не разрешить его просмотр.

Для каждого поля из списка разрешенных вы также можете указать определенные модификаторы. Так, например, вы можете разрешить изменение поля не всем пользователям с этой ролью, а только тому, кто является автором или ответственным задачи. Для того, чтобы использовать модификатор, выберите нужное поле (или несколько полей) из списка и нажмите кнопку модификатора. Вы можете использовать два модификатора (**Автор** и **Ответственный**). Пары модификаторов работают по **ИЛИ**.

В данном случае все модификаторы действуют для всех правил одинаково, то есть **Автор** — это автор задачи, а **Ответственный** — ответственный за нее на момент проверки прав.
