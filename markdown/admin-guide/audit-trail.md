# Как отследить историю изменения параметров задачи

## Что такое аудит изменений и зачем он нужен

В TrackStudio вы можете отслеживать все изменения, которые происходят с задачей и контролировать изменившиеся данные. Эта функциональность реализуется через операцию специального типа, соответственно вы можете включить отслеживание изменений для задач с определенным процессом, а также гибко управлять доступом к просмотру этих изменений с помощью обычного механизма контроля доступа в TrackStudio.

## Как включить отслеживание изменений

1. Перейдите через меню TrackStudio к нужному вам процессу
2. Перейдите к списку операций
3. Добавьте операцию с названием "*" (один символ)
4. Перейдите в этой операции к вкладке "Разрешение для операций".
5. Задайте правила просмотра операции

Никаких переходов, других правил доступа, триггеров, резолюций и т.п. устанавливать не нужно. Аудит изменений работает на уровне ядра TrackStudio, поэтому триггеры не будут выполняться, а правила доступа - проверяться.

## Как управлять видимостью аудита

Однако при просмотре операций для аудита изменений применимы те же правила доступа, что и для обычных операций. То есть в настройках правил вы можете указать, пользователи с какими ролями смогут видеть записи аудита, а с какими - не могут.

## Как это выглядит

В поставляемой с TrackStudio базой данных "Управление требованиями" аудит изменений уже включен для процессов "**Ошибка**" и "**Набор тестовых данных**". По-умолчанию записи аудита видят только администраторы.

Просмотреть записи аудита можно на странице просмотра задачи. Для этого нужно нажать кнопку "**Показать аудит**" на панели операций.
