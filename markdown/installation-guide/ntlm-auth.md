# Настройка авторизации через NTLM

Чтобы настроить авторизацию пользователей через NTLM:

1. Войдите в **Microsoft Windows**** **как Administrator.
2. Запустите **Configure Your Server** (Настройка сервера). (**Панель управления / Администрирование / Настройка сервера**).
3. На вкладке **Networking/DHCP** настройте сервис DHCP, если требуется.
4. На вкладке **Networking/DNS** настройте DNS, если требуется.
5. На вкладке **Active Directory** настройте контроллер домена.
6. Для установки WINS используйте мастер **Windows Components**
7. Включите NTLM в файле **trackstudio.security.properties**

```
trackstudio.useNTLM yes
```

1. Укажите имя домена и адрес WINS в **trackstudio.security.properties**

```
jcifs.smb.client.domain=WORKGROUP
jcifs.netbios.wins=192.168.100.1
```

1. Нажмите на кнопку **Проверить соединение**

Как это работает:

Если **trackstudio.useNTLM** установлено в **yes**, TrackStudio будет использовать NTLM для авторизации пользователей. Эта схема авторизации позволяет сохранять данные об авторизации пользователей с помощью сервиса WINS и, таким образом, освобождает пользователей от необходимости вводить логин и пароль всякий раз, когда они с одного рабочего места заходят в TrackStudio.

Примечание

- Механизм NTLM использует сервисы WINS и DNS, поэтому в вашей сети должен быть хост, настроенный как контроллер домена Windows. Контроллер домена может быть установлен только под серверной версией ОС Windows.
- Если вы используете NTLM авторизацию, учетную запись в TrackStudio все равно нужно создавать. Логин при этом должен совпадать с логином в Windows.
