# Как с помощью TrackStudio учитывать рабочее время

С помощью TrackStudio вы можете учитывать свое рабочее время, затраченное на решение задач, и рабочее время своих сотрудников.

TrackStudio не следит за деятельностью пользователей и не делает экранные снимки. Пользователи самостоятельно подсчитывают затраченное на решение задачи время и вносят его при выполнении очередной операции над задачей. Например,

- Пользователь исправляет задачу и хочет отразить это в системе. При этом он выполняет операцию "resolve", меняет ответственного на тестировщика, добавляет комментарий и вводит время потраченное на задачу (например, 1 час 20 минут). Все это выполняется на одной форме. Для упрощения подсчета времени в каждой задаче TrackStudio для каждого пользователя может инициироваться индивидуальный таймер. Он считает время начиная с момента загрузки страницы, что удобно для коротких задач, характерных для Service Desk: регистрации инцидентов, ответа на телефонный звонок и т.п.

- Тестировщик тестирует, добавляет сообщение типа "verify" и указывает свое время (например, 40 минут).

- Менеджер смотрит на задачу и видит время, указанное в операциях каждым разработчиком, общее время всех разработчиков в свойствах задачи.

- Менеджер, или другой пользователь, может посмотреть, сколько времени ушло на проект, в свойствах проекта: там выводится именно суммарное время по всем подзадачам проекта. Для проекта может быть выбран [другой формат вывода времени](categories/manage-categories.md), например в днях или месяцах.

- Также доступна разнообразная статистика: например, "сколько времени потратили разработчики на приоритетные задачи за последнюю неделю", "на какой проект ушло больше всего времени за последний год", "сколько времени было потрачено на тестирование задач") — можно [построить отчет](howto-make-reports/index.md) и информация будет в любом удобном формате: CSV, HTML, PDF, XML с XSL-преобразованием и т.п. Есть и экспорт в MS Project. Обратите внимание, что время всех разработчиков не суммируется в одном поле, можно посчитать время по каждому человеку и каждой операции (verify/close/resolve/etc).

- С помощью подписки можно получать периодическую статистику на электронную почту; например, список внесенных разработчиками часов за день ежедневно в 18 часов.

- Для каждой задачи или проекта менеджер может установить "**Бюджет**", а потом искать задачи (или получать особое оповещение), если бюджет превышен.

- Для каждого пользователя можно указать для каких проектов он может смотреть или редактировать внесенное время, а для каких нет.
